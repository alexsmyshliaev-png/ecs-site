# Запуск проекта локально

## Фронтенд (статика)

```bash
cd "ЕЦС сайт"
python3 -m http.server 8765 --directory frontend
# открыть http://localhost:8765
```

В Claude Code оба сервера описаны в `.claude/launch.json` (`ecs-site`, `ecs-api`).

## Бэкенд (FastAPI)

```bash
cd "ЕЦС сайт/backend"
python3 -m venv .venv               # один раз
.venv/bin/pip install -r requirements.txt
cp .env.example .env                # и заполнить значения
.venv/bin/python -m uvicorn app.main:app --port 8000 --reload
# Swagger: http://localhost:8000/docs
```

Чтобы фронтенд слал лиды и события в бэкенд, в
`frontend/assets/js/config.js` укажите `apiUrl: "http://localhost:8000"`.

## Тесты

```bash
cd backend && .venv/bin/python -m pytest tests/ -q
```

## Переменные окружения (backend/.env)

| Переменная | Что это |
|---|---|
| `BITRIX_WEBHOOK_URL` | Входящий вебхук Битрикс24 (права CRM). Пусто — лиды копятся только в SQLite |
| `ADMIN_TOKEN` | Токен для чтения лидов/событий (`Authorization: Bearer <токен>`). Пусто — чтение закрыто |
| `DATABASE_URL` | По умолчанию `sqlite:///./ecs.db` |
| `CORS_ORIGINS` | JSON-список адресов фронтенда |

## Настройки фронтенда (frontend/assets/js/config.js)

Один файл со всеми интеграциями: телефон/почта/адрес, `apiUrl`,
виджет Битрикс24, Яндекс.Метрика, GA4, ключ Яндекс.Карт + координаты офиса,
`data-id` калькуляторов Финуслуг (osago / travel), ссылки на мессенджеры.

## Пересборка страниц

Шапка, меню, футер и модалка — общие. Меняются в `frontend/index.html`,
затем разносятся по остальным страницам генератором:

```bash
cd frontend && python3 build_pages.py
```

(index.html генератор не трогает — это источник общих блоков.)

## Переслать сайт человеку (без хостинга)

Обычные .html из frontend/ нельзя пересылать поштучно — стили и картинки
лежат рядом в assets/, и у получателя страница откроется «голой».
Вместо этого соберите автономную версию:

```bash
cd frontend && python3 build_share.py
```

Появятся `dist/*.html` (каждый файл самодостаточен: CSS, JS и картинки
встроены внутрь) и архив `ЕЦС-сайт-демо.zip` (~2 МБ) — его и отправляйте.
Получатель распаковывает и открывает любой .html двойным кликом; переходы
между страницами работают, пока файлы лежат в одной папке.

Не работает в пересланной версии (нужен настоящий хостинг): калькуляторы
Финуслуг/polis812, виджет Битрикс24, Яндекс.Карты; шрифт Inter подтянется
только при интернете.

## Подключение Битрикс24 (CRM)

Изучено по официальной документации: <https://apidocs.bitrix24.com>
(репозитории [bitrix24-dev-hub](https://github.com/bitrix24/bitrix24-dev-hub) и
[b24restdocs](https://github.com/bitrix24/b24restdocs)).

### 1. Входящий вебхук (лиды из форм сайта)

1. В портале: **Приложения → Разработчикам → Готовые сценарии → Другое →
   Входящий вебхук**.
2. В правах доступа отметить **CRM (crm)**.
3. Скопировать URL вида `https://ваш-портал.bitrix24.ru/rest/1/секрет/`
   и вписать в `backend/.env` → `BITRIX_WEBHOOK_URL`.
4. Перезапустить бэкенд — лиды из всех форм начнут создаваться в CRM.

Лид создаётся методом **`crm.item.add`** (`entityTypeId: 1`) — прежний
`crm.lead.add` в документации помечен устаревшим. Поля передаются в
camelCase (`title`, `comments`, `sourceId`, `utmSource`…), телефон и email —
массивом мультиполей `fm`. Вместе с лидом уходят UTM-метки, страница
и название формы.

Вебхук бессрочный и секретный: попадание URL наружу даёт полный доступ
к CRM в рамках его прав. Поэтому боевой сценарий — только через бэкенд
(`apiUrl` в config.js); `bitrixWebhookUrl` на фронтенде — временный
вариант без бэкенда, вебхук при этом виден любому посетителю.

### 2. Виджет на сайт (чат, обратный звонок, CRM-формы)

Битрикс24 → **CRM → Интеграции → Виджет на сайт**, скопировать URL
загрузчика (`.../site_button/loader_X_xxxxxx.js`) в `bitrixWidgetUrl`
(config.js). Кода больше не требуется — загрузчик уже в `bitrix.js`.

### 3. Лимиты API

Облачный Битрикс24 ограничивает интенсивность (leaky bucket, ~2 запроса/сек
устойчиво, всплески допустимы; при превышении — HTTP 503
`QUERY_LIMIT_EXCEEDED`). Для потока заявок с сайта это с запасом.

## Что заменить перед запуском в прод

1. Телефон `8 800 123-45-67`, почта `info@ecs.ru`, адрес офиса — заглушки.
2. Фото: `frontend/travel.html` (блок `.ph` в hero) и `frontend/office.html` (галерея `.ph`).

5. Проверить калькуляторы Финуслуг на боевом домене (на localhost они не работают).
