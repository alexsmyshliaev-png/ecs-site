# UI и бренд ЕЦС — правила текстом

Извлечено из «ЕЦС — Руководство по бренду.pdf» (файл внутренний, в git не хранится) и
из уже принятых решений в CSS. Значения — токены `frontend/assets/css/tokens.css`;
хардкодить hex в `components.css`/`base.css` нельзя.

## Цвет
- Основной красный — **`#e60023`** (токен `--red`, он же `--accent`). Оттенки:
  `--red-deep #d0222c`, `--red-700 #b3001c` (ховер кнопок).
- **Одно главное действие на экран.** Красная заливка (`.btn--primary`) — единственная
  на видимой области. Вторичные действия — `.btn--ghost`/`.btn--outline`. Не ставить
  две красные кнопки рядом.
- Красная тень/акцент — сдержанно: подчёркивает главную кнопку и бейдж логотипа, не украшение.

## Логотип
- Логотип ЕЦС **не изменять** (пропорции, цвета, элементы). Допустима только компоновка/размер.
- Тёмная/светлая версии: `.brand__logo--light` / `--dark` переключаются темой автоматически — не дублировать вручную.
- Логотипы страховых-партнёров: по умолчанию **grayscale, opacity ~0.78**
  (`.logo` в `components.css`). В маркизе при ховере одного лого остальные приглушаются
  (opacity .55), наведённое — в полном цвете. Это бренд-правило, не менять на «всегда цветные».

## Шрифт
- **Inter** (self-hosted, `assets/fonts/`, `@font-face` в `tokens.css`, `font-display: swap`,
  preload cyrillic+latin). Google Fonts не подключать — CDN нестабилен для РФ и бьёт по LCP.

## Компоненты и токены
- Радиусы, тени, поверхности, границы — только токены (`--radius*`, `--shadow*`, `--surface`,
  `--border*`). Темы `clean`/`bold`/`dark` переключаются через `data-theme`.
- Кнопки: `.btn--primary` (красная, одна на экран), `.btn--ghost`, `.btn--outline`, размер `.btn--lg`.
- Заглушки фото — `.ph`; карточки — `.card`/`.card--feature`/`.card--accent`.

## Доступность и разметка
- Один `<h1>` на страницу; `alt` у всех `<img>`; у изображений — явные `width`/`height` (против CLS).
- Ссылки-иконки/кнопки без текста — `aria-label`.
- Пользовательский ввод не вставляется через `innerHTML`.

Соглашения по data-атрибутам и JS-модулям — в `docs/frontend/components.md`.
