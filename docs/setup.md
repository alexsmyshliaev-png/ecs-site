# Запуск проекта локально

Сайт — чистая статика (ADR 2026-07-07, см. architecture.md). Собственного
бэкенда нет: заявки принимают CRM-формы Битрикс24. Достаточно раздать `frontend/`.

## Фронтенд (статика)

```bash
cd "ЕЦС сайт"
python3 -m http.server 8765 --directory frontend
# открыть http://localhost:8765
```

В Claude Code сервер описан в `.claude/launch.json` (`ecs-site`).

> Прежний FastAPI-бэкенд выведен из эксплуатации и лежит в
> `archive/fastapi-backend/` (там свой README) — для локальной работы не нужен.

## Настройки фронтенда (frontend/assets/js/config.js)

Один файл со всеми интеграциями: телефон/почта/адрес, `b24forms` (id/code/loader
CRM-форм Битрикс24), виджет Битрикс24, Яндекс.Метрика, GA4, ключ Яндекс.Карт +
координаты офиса, `data-id` калькуляторов Финуслуг (osago / travel), ссылки на
мессенджеры.

## Пересборка страниц

Шапка, меню, футер и модалки — общие. Меняются в `frontend/index.html`,
затем разносятся по остальным страницам генератором:

```bash
cd frontend && python3 build_pages.py
```

(index.html генератор не трогает — это источник общих блоков.)

## Переслать сайт человеку (без хостинга)

Обычные .html из frontend/ нельзя пересылать поштучно — стили и картинки
лежат рядом в assets/, и у получателя страница откроется «голой».
Вместо этого соберите автономную версию:

```bash
cd frontend && python3 build_share.py
```

Появятся `dist/*.html` (каждый файл самодостаточен: CSS, JS и картинки
встроены внутрь) и архив `ЕЦС-сайт-демо.zip` (~2 МБ) — его и отправляйте.

Не работает в пересланной версии (нужен настоящий хостинг): калькуляторы
Финуслуг/polis812, CRM-формы и виджет Битрикс24, Яндекс.Карты; шрифт Inter
подтянется только при интернете.

## Подключение Битрикс24 (CRM-формы + виджет)

Документация (сверяться с ней, не с памятью):
- Размещение CRM-формы на стороннем сайте: <https://helpdesk.bitrix24.ru/open/13003062/>
- Работа с кодом форм (скрытые поля, `b24:form:init`): <https://helpdesk.bitrix24.ru/open/12853296/>
- Скрытые поля: <https://helpdesk.bitrix24.ru/open/25978110/> · UTM: <https://helpdesk.bitrix24.ru/open/6266591/>
- Антиспам форм: <https://helpdesk.bitrix24.ru/open/24938078/>
- REST: <https://apidocs.bitrix24.ru/> · Хаб: <https://github.com/bitrix-tools/bitrix24-dev-hub>

### 1. CRM-формы (приём заявок) — без сервера и без секретов

1. Портал: **CRM → CRM-формы** → создать формы (минимум «Заявка» и «Запись в офис»).
   Скрытые поля: `product`, `form_slug`, `page_url`, `utm_first`, `referrer_first`,
   `ym_client_id`, `plate`. Включить SmartCaptcha, поле согласия (ссылка на
   `privacy.html`), автозахват UTM.
2. У формы → **«Ссылка и код» → «Скрипт»** — скопировать embed вида
   `<script data-b24-form="inline/ID/CODE" ...>(…)(…, 'https://cdn-ru.bitrix24.ru/bXXXX/crm/form/loader_N.js');</script>`.
3. В `config.js` → `b24forms.lead` / `b24forms.office` вписать:
   `id` и `code` из `inline/ID/CODE`, `loader` — URL `…/crm/form/loader_N.js`.

Пока поля пустые — на месте формы показывается фолбэк «позвоните/напишите»
(`leads.js`). Скрытые поля наполняет `leads.js` по событию `b24:form:init`.
Секретов на фронте нет: id/code формы публичны, доступ к CRM не дают —
поэтому вебхук больше не нужен.

### 2. Виджет на сайт (чат, обратный звонок)

Битрикс24 → **CRM → Интеграции → Виджет на сайт**, скопировать URL загрузчика
(`.../site_button/loader_X_xxxxxx.js`) в `bitrixWidgetUrl` (config.js). Кода
больше не требуется — загрузчик уже в `bitrix.js`.

## Что заменить перед запуском в прод

1. `config.js` → `b24forms.lead` / `b24forms.office` (id/code/loader форм из портала).
2. Телефон `8 800 123-45-67`, почта `info@ecs.ru`, адрес офиса — заглушки.
3. Фото: `frontend/travel.html` (блок `.ph` в hero) и `frontend/office.html` (галерея `.ph`).
4. Ключ Яндекс.Карт + координаты офиса в config.js — карта на office.html включится сама.
5. `data-id` калькулятора путешествий (`calc.travel` в config.js).
6. Проверить калькуляторы Финуслуг на боевом домене (на localhost они не работают).
