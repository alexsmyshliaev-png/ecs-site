# Архитектура

## Стек и почему

| Слой | Технология | Почему |
|---|---|---|
| Frontend | Статические HTML + ванильный JS-модули | Страницы свёрстаны по бренд-гайду, мгновенная загрузка, хостинг на чём угодно. React не используется осознанно (ADR 2026-06-11) |
| Приём заявок | CRM-формы Битрикс24 (виджет) | Лид создаётся в CRM напрямую, без сервера и без секретов в браузере; встроены капча (SmartCaptcha), согласие 152-ФЗ, автозахват UTM (ADR 2026-07-07) |
| Аналитика | Яндекс.Метрика + GTM | Метрика для маркетинга и целей (reachGoal); прочие теги — через GTM. Собственной БД событий нет |
| Backend | — (архивирован) | FastAPI-бэкенд выведен из эксплуатации 2026-07-07, лежит в `archive/fastapi-backend/` |

## Поток лида

```
Слот CRM-формы на странице  <div class="b24form" data-b24form="lead" ...>
  └─ leads.js монтирует embed формы Битрикс24 из config.b24forms
        ├─ b24:form:init → form.setProperty(): product, form_slug, page_url,
        │                   utm_first, referrer_first, ym_client_id, plate
        ├─ отправка формы → лид создаётся В CRM напрямую (без нашего сервера)
        │                   события транслируются в ecsTrack (lead_form_open, lead_success)
        └─ форма не настроена / CDN недоступен (таймаут 5 с)
                          → фолбэк «позвоните/напишите» + ecsTrack("b24form_failed")
```

Скрытые поля наполняются из UTM первого/последнего касания и реферера (utm.js).
ПДн в localStorage не пишутся (см. ADR 2026-07-07).

## Поток аналитики

`ecsTrack(event, params)` (analytics.js) рассылает событие в два места:
dataLayer (GTM) и Яндекс.Метрику (reachGoal). Разметка целей — атрибут
`data-track="имя_цели"` на любом элементе. Собственного `/api/events` больше нет.

## Структура

```
frontend/                — 9 HTML-страниц + assets (css: tokens/base/components, js: 7 модулей)
archive/fastapi-backend/ — выведенный из эксплуатации FastAPI-бэкенд (справочно)
docs/                    — этот каталог
```

## Принятые решения (ADR)

1. **2026-06-11 · Статический фронтенд вместо React** — сайт уже работал, виджеты
   (Финуслуги, Битрикс24, Метрика) нативно встраиваются в HTML; пользователь подтвердил выбор.
2. **2026-07-07 · Отказ от собственного бэкенда в пользу CRM-форм Битрикс24** —
   для прода бэкенду нужны VPS, деплой, rate limiting, ретраи, мониторинг, бэкапы БД
   с ПДн: несоразмерный ops-объём для корпоративного сайта. CRM-формы закрывают всё то же
   без сервера: лид в CRM напрямую, встроенные капча/согласие/UTM, никаких секретов
   в браузере. Каталог `backend/` заархивирован в `archive/fastapi-backend/`.
   Критерии возврата: свой личный кабинет, серверные расчёты, интеграции сверх CRM,
   собственное хранилище событий для BI. (Отменяет ADR 2026-06-11 «лид сначала в свою БД».)
3. **Бренд-гайд как закон** (ЕЦС — Руководство по бренду.pdf): красная заливка —
   только одно главное действие на экран; красная тень — только под бейджем логотипа;
   логотип менять нельзя (только компоновку); партнёрские логотипы grayscale ~78%.

## Ограничения

- Калькуляторы Финуслуг привязаны к домену партнёра — на localhost не отрисовываются
  (стоит фолбэк «Получить расчёт от менеджера», событие `calc_widget_failed`).
- CRM-формы Битрикс24 рендерятся с CDN `*.bitrix24.ru`; на localhost без настроенных
  `config.b24forms.*` показывается фолбэк «позвоните/напишите».
