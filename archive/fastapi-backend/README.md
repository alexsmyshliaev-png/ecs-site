# Архив: FastAPI-бэкенд ЕЦС

> **Не используется в проде.** Каталог выведен из эксплуатации 2026-07-07
> и оставлен как справочный материал/история.

## Почему заархивирован

Бэкенд существовал ради двух задач: (1) скрыть вебхук Битрикс24 от браузера
при отправке лидов и (2) вести собственную БД событий аналитики. Аудит
(`docs/AUDIT-2026-07-07.md`) показал, что для прода это несоразмерный
ops-объём (VPS, деплой, rate limiting, ретраи, мониторинг, бэкапы БД с ПДн).

Решение (ADR 2026-07-07, см. `docs/architecture.md`): заявки принимаются
**CRM-формами Битрикс24** (лид создаётся в CRM напрямую, без сервера и без
секретов в браузере), аналитика — Яндекс.Метрика. Сайт стал чистой статикой.

## Когда вернуться к бэкенду

Критерии из ADR: свой личный кабинет клиента, серверные расчёты, интеграции
сверх CRM, потребность в собственном хранилище событий для BI.

## Как запустить (если понадобится)

```bash
cd archive/fastapi-backend
python3 -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # заполнить
uvicorn app.main:app --reload
```
