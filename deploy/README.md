# Публикация сайта ЕЦС

Сайт — чистая статика (`frontend/`). Нужен хостинг **с поддержкой HTTP-заголовков**
(GitHub Pages их не умеет; калькуляторы Финуслуг ещё и привязаны к домену партнёра).

## Варианты хостинга

| Вариант | Заголовки | Как применить |
|---|---|---|
| **Cloudflare Pages** | файл `frontend/_headers` | подключить репозиторий, publish dir = `frontend` |
| **Netlify** | файл `frontend/_headers` | то же |
| **VPS + nginx** | `deploy/nginx-security-headers.conf` | `include` в server-блоке |

Для РФ-аудитории подойдёт российский статик-хостинг/VPS (рег.ру и аналоги) —
тогда используется nginx-вариант.

## Порядок запуска

1. Раздать содержимое `frontend/` на боевом домене по HTTPS.
2. Заголовки: взять `frontend/_headers` (Pages/Netlify) или
   `deploy/nginx-security-headers.conf` (nginx).
3. **CSP — сначала в режиме отчёта.** На 1–2 дня включить
   `Content-Security-Policy-Report-Only`, проверить, что живы: CRM-формы и виджет
   Битрикс24, калькуляторы ОСАГО/ипотека/путешествия, Яндекс.Карта на «Офисе».
   Затем переключить на enforce.
4. Проверить оценку на <https://securityheaders.com> — цель **A**.
5. `HSTS` включать только когда весь трафик уже на HTTPS.

## github.io / старый адрес

После переезда закрыть старую версию от индексации:
- либо отключить GitHub Pages,
- либо добавить в деплой Pages `<meta name="robots" content="noindex">` и
  вернуть заголовок `X-Robots-Tag: noindex` только для домена `*.github.io`.

## Что перепроверить на боевом домене

- Калькуляторы Финуслуг/polis812 рендерятся (на localhost — нет, привязаны к домену).
- CRM-формы Битрикс24 (`config.b24forms`) отдаются с `*.bitrix24.ru` — если CSP
  ругается, добавить точный домен портала в `script-src`/`frame-src`.
- Метрика грузится только после согласия (cookie-баннер).
