# Архитектура

## Стек и почему

| Слой | Технология | Почему |
|---|---|---|
| Frontend | Статические HTML + ванильный JS-модули | Страницы уже свёрстаны по бренд-гайду, мгновенная загрузка, хостинг на чём угодно. React не используется осознанно (решение от 2026-06-11): переписывание не давало ценности, структура скилла применена к модулям JS |
| Backend | FastAPI + SQLAlchemy + SQLite | Лёгкий API для лидов и аналитики; SQLite достаточно для старта, миграция на Postgres — заменой `DATABASE_URL` |
| CRM | Битрикс24 | Вебхук `crm.lead.add` вызывается с сервера — секрет не попадает в браузер |
| Аналитика | Яндекс.Метрика + GA4 + собственная (events) | Метрика для маркетинга, собственная БД — для сквозных отчётов «событие → лид» |

## Поток лида

```
Форма на странице (leads.js)
  └─ POST {apiUrl}/api/leads  ──── сохранён в SQLite (всегда)
        │                          └─ переслан в Битрикс24 (bitrix_service, статус в bitrix_status)
        ├─ фолбэк 1: прямой вебхук Битрикс24 из config.js (если бэкенда нет)
        └─ фолбэк 2: localStorage "ecs_leads" (если нет вообще ничего)
```

В каждый лид вшиваются UTM первого и последнего касания, реферер и страница (utm.js).

## Поток аналитики

`ecsTrack(event, params)` (analytics.js) рассылает событие в три места:
dataLayer (GTM/GA4), Метрику (reachGoal), и `POST /api/events` через sendBeacon.
Разметка целей — атрибут `data-track="имя_цели"` на любом элементе.

## Структура

```
frontend/   — 6 HTML-страниц + assets (css: tokens/base/components, js: 7 модулей)
backend/    — FastAPI (routers → services → models), тесты, .venv
docs/       — этот каталог
```

## Принятые решения (ADR)

1. **2026-06-11 · Статический фронтенд вместо React** — сайт уже работал, виджеты
   (Финуслуги, Битрикс24, Метрика) нативно встраиваются в HTML; пользователь подтвердил выбор.
2. **2026-06-11 · Лид сначала в свою БД, потом в CRM** — заявка не теряется
   при недоступности Битрикс24; статус пересылки хранится в `bitrix_status`
   (pending/sent/failed/disabled).
3. **Бренд-гайд как закон** (ЕЦС — Руководство по бренду.pdf): красная заливка —
   только одно главное действие на экран; красная тень — только под бейджем логотипа;
   логотип менять нельзя (только компоновку); партнёрские логотипы grayscale ~78%.

## Ограничения

- Калькуляторы Финуслуг привязаны к домену партнёра — на localhost не отрисовываются
  (стоит фолбэк «Получить расчёт от менеджера», событие `calc_widget_failed`).
- Python на машине разработки — 3.9: в бэкенде используется `Optional[X]`, не `X | None`.
